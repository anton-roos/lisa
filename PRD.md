# **Project Requirements Document: DCEG Learner Information System Phase 1 and 2**

The following table outlines the detailed functional requirements for the DCEG Learner Information System:

| Requirement ID | Description                        | User Story                                                                                                                                               | Expected Behavior/Outcome                                                                                                       |
|----------------|------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------|
| FR001           | Progress Feedback                  | As a user, I want to select a subject and grade, view a table of learners, and optionally print the information.                                            | The system should allow subject and grade selection, display learners in a table, and offer a print facility (non-critical).   |
| FR002           | Attendance: Select Class           | As a user, I want to select a grade and class list (or subject combinations) to manage attendance.                                                         | The system should allow selection of grade, then class list or subject in combination, or pillar/care group.                  |
| FR003           | Attendance: Mark Learners Present  | As a teacher, I want to mark learners as present using an easy bullet-type interface.                                                                     | A bullet-type attendance marking option should be available after selecting the period and start time.                       |
| FR004           | Attendance: Lock Attendance        | As a teacher, I want the system to lock attendance after a set period to ensure punctuality.                                                              | Attendance should auto-lock either manually via button or automatically three minutes after the bell rang.                    |
| FR005           | Attendance: Late Learner Handling  | As a teacher, I want to mark late learners and record how many minutes they were late.                                                                     | The system should allow marking learners late, automatically calculating minutes late based on time entered.                  |
| FR006           | Late for School Attendance         | As a teacher, I want to record the exact arrival time for learners who arrive late to school.                                                              | Provide a field to capture "time arrived" for school-wide late attendance records.                                             |
| FR007           | Behaviour Tracking                 | As a teacher or administrator, I want to record learner behavior incidents.                                                                                | The system should include a simple module to log and track behavior issues.                                                    |
| FR008           | Quality Assurance Control (QAC)    | As school management, I want access to a list of QA reports regarding learner performance, attendance, and behavior.                                        | A section should exist for predefined reports aligned to QA needs.                                                            |
| FR009           | Management Reports: Performance    | As school management, I want to generate performance reports based on specific criteria (e.g., marks below/above thresholds).                              | Reports per learner displaying marks across subjects based on selected thresholds (e.g., <40%, 40–60%, >60%, >80%).           |
| FR010           | Management Reports: Absenteeism    | As school management, I want to view reports on learner absenteeism.                                                                                       | Generate reports for: learners absent for a day, absent more than X days, present at school but missing classes/care groups.   |
| FR011           | Attendance List per Learner        | As school management, I want to see detailed attendance per learner over time.                                                                             | Reports showing: school absences, late days, subjects missed while present, and care group attendance.                        |
| FR012           | WA Communication                   | As administrators, I want a communication channel (WA = WhatsApp) to be integrated for messaging purposes.                                                 | Set up basic WA communication capabilities for relevant use cases.                                                            |
| FR013           | Time Scheduling for Progress Feedback | As school management, I want to set schedules for progress feedback reporting.                                                                              | Configure time schedules for when progress feedback must be captured and reported.                                            |
| FR014           | Results Display Phase 1            | As a teacher, I want to see my test results per subject and grade.                                                                                          | Show teacher test results at the subject and grade level. If too many issues arise, defer to phase two.                       |
| FR015           | Communication Module Enhancement   | As an administrator, I want to create and edit communication templates.                                                                                   | DK to design and edit templates; prioritize testing critical templates first, others can follow later.                        |
